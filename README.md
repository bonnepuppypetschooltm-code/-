# 犬の幼稚園 送迎ルート自動化ツール (bonnepuppey天満店)

Google カレンダーの予定から送迎対象の園児を抽出し、クレート(車載数)制限を
考慮して送迎ルートの一覧表を作成します。各便には、そのままタップすれば
Googleマップでナビが開くリンクが付きます(APIキー・クレジットカード登録は不要です)。

## カレンダーの記法ルール

送迎が必要な園児の予定タイトルの先頭に `🚗` を付けてください。

```
🚗 ポチ [大]
🚗 ポチ [往復 大]
🚗 ポチ [迎えのみ 中]
🚗 ポチ [送りのみ 小]
🚗 ポチ [迎えのみ 9:00 特大]
🚗 中前大地 [朝のみ 8:00 大]
🚗 中前大地 [夕のみ]
🚗 ロイ [往復 8:00 17:30 大]
🚗 桐野ロイ [往復 朝8:00 大]
```

- タグ省略 / `[往復]` : 朝の「お迎え便」と夕方の「送り便」の両方の対象になります
- `[迎えのみ]` / `[朝のみ]` : 朝の「お迎え便」のみ対象
- `[送りのみ]` / `[夕のみ]` : 夕方の「送り便」のみ対象
- タグ内に `9:00` のように時刻を書くと、その時刻を希望時刻として優先的に並び替えます
  (時刻指定がない場合は予定の開始時刻が使われます)
- 往復で朝と夕方の希望時刻が違う場合は、`[往復 8:00 17:30]` のように
  時刻を2つ並べて書くと、1つ目が朝、2つ目が夕方の希望時刻になります
- 往復で**片方だけ**希望時刻がある場合は、`[往復 朝8:00]` / `[往復 夕17:30]` のように
  時刻の直前に `朝` または `夕` を書いてください
  (指定がない方の便は、時刻指定がないお宅として扱われます)
- タグ内に `特大` `大` `中` `小` のいずれかを書くと、その子に使うクレートのサイズになります
  (指定がない場合は `config.yaml` の `default_crate_size` が使われます)

**注意**: `[朝のみ 8:00 大]` のように、上記の例と同じ書き方をしてください。
`[朝8時まで 夕指定なし大]` のような自由な書き方では正しく認識されません。

`[往復 朝8:00 夕17:00] [大]` のように `[ ]` を分けて書いても大丈夫です
(まとめて1つのタグとして読み取られます)。

「場所」欄には、その子の**自宅住所**を入力してください。
住所の先頭に郵便番号(`〒532-0013` など)が付いていても問題ありません。

## 訪問順について

- **時刻指定があるお宅**: その時刻順を優先します
- **時刻指定がないお宅**: 店舗から距離が遠い順に並びます(遠方を先に回り、最後は店舗の近くで締めくくる順番)
  - 距離は「国土地理院 住所検索API」(無料・APIキー不要)を使って自動計算します
  - 一度調べた住所は `.geocode_cache.json` に保存され、次回以降は高速になります

## 出発時刻について

各便の「出発」時刻は、以下のルールで自動的に決まります。

- その便に時刻指定のあるお宅が**ある**場合: 一番早い希望時刻から
  `departure_buffer_minutes`(分)前を出発時刻にします
  (例: 一番早い希望時刻が 8:00、`departure_buffer_minutes: 15` なら 7:45 出発)
- 時刻指定のあるお宅が**ない**場合: `config.yaml` の
  `morning_start_time` / `evening_start_time` を出発時刻にします

毎日カレンダーの内容にあわせて自動調整されるため、`config.yaml` を
日々編集する必要はありません。

## 帰着予定・移動時間(目安)について

各便の表に「帰着予定」として、店舗に戻ってくる時刻の目安を表示します。
また、表の各行には以下も表示されます。

- 「ここまで」: **ひとつ前の場所**(店舗、または前のお宅)からそのお宅までの移動時間の目安
- 「次まで」: そのお宅から**次の目的地**(次のお宅、または最後のお宅なら店舗)までの移動時間の目安

これらは住所間の距離・件数をもとに計算した**あくまで目安**で、
実際の道路状況により前後します。住所の位置情報が取得できない場合は「-」と表示されます。

実際にかかる時間と比べて短すぎる/長すぎる場合は、`config.yaml` の以下の値で調整してください。

- `avg_speed_kmh`(平均時速、既定20km/h): 表示時間が**短すぎる**場合は下げてください
- `route_distance_factor`(道なり距離の補正係数、既定1.3): 表示時間が**短すぎる**場合は上げてください
  (地図上の直線距離に対し、実際の道路はこの倍だけ長いとみなして計算します)
- `stop_minutes`(1軒あたりの乗せ降ろし時間、既定5分)

### 店舗とお宅の間で、実際にかかる時間が決まっている場合

「店舗→桐野ロイは15分だけど、桐野ロイ→店舗は5分」のように、
行き帰りで時間が違う・実際の時間がすでにわかっているお宅は、
以下のコマンドで登録できます(`config.yaml` を手で編集する必要はありません)。

```bash
python route_planner.py --date 2026-06-13 --set-travel-time "桐野ロイ" --from-store 15 --to-store 5
```

- `--date` で指定した日のカレンダーから、その名前の🚗予定の住所を自動的に取得します
- `--from-store`: 店舗 → そのお宅 までの時間(分)
- `--to-store`: そのお宅 → 店舗 までの時間(分)
- どちらか片方だけ指定してもOKです(指定しなかった方は通常の計算結果が使われます)
- 実行すると `config.yaml` の `travel_time_overrides` に自動で追記されます

## クレート(車載数)の制限について

その日に積むクレートのサイズの組み合わせはバラバラ(例: 大2個+中6個)になるため、
`config.yaml` の `crate_capacity` で「そのサイズ**だけ**で車を満載にした場合の最大数」を
設定しておくと、サイズが混在していても自動で「車に収まる組み合わせ」ごとに便を分けます。

```yaml
crate_capacity:
  特大: 2
  大: 4
  中: 9
  小: 12
default_crate_size: "中"
```

出力画面には、各便ごとに「積載: ◯◯」「あとまだ積めます: ◯◯」のように、
クレートの個数で分かりやすく表示されます。

## まずは試してみる(API設定不要)

サンプルデータを使って、出力イメージをすぐに確認できます。
Google CalendarやMaps APIの設定は一切不要です。

```bash
pip install pyyaml
python3 route_planner.py --demo
```

「やりたいこと」のイメージと合っているか確認してから、本番セットアップ(下記)に進んでください。

## セットアップ

1. 依存パッケージをインストール

   ```bash
   pip install -r requirements.txt
   ```

2. Google Calendar API の認証情報を取得(無料・クレジットカード不要)

   - [Google Cloud Console](https://console.cloud.google.com/) でプロジェクトを作成
   - 「Google Calendar API」を有効化
   - 「OAuth クライアント ID」(デスクトップアプリ)を作成し、`credentials.json` としてこのディレクトリに保存

3. `config.example.yaml` を `config.yaml` としてコピーし、拠点住所・クレート数・出発時刻を編集

   ```bash
   cp config.example.yaml config.yaml
   ```

## 実行方法

```bash
python route_planner.py
```

特定の日付を指定する場合:

```bash
python route_planner.py --date 2026-06-15
```

初回実行時はブラウザが開き、Google アカウントでの認証を求められます。
認証後は `token.json` に保存され、以後は自動で再利用されます。

## 出力

実行すると `送迎ルート_YYYY-MM-DD.html` という表形式のファイルが作成され、
自動的にブラウザで開きます(便ごとに「出発時刻」「積載状況」「順番・名前・住所・
希望時刻」の表、「Googleマップでルートを開く」ボタン、画面に埋め込まれた地図)。

- 出力先を変えたい場合: `--output 保存先.html`
- ブラウザで自動的に開かないようにする場合: `--no-open`
